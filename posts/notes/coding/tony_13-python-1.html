9. Classes

Classes provide a means of bundling data and functionality together.

Features include:

  • inheritance: allowing multiple base classes
  • created at runtime, can be modified after creation
  • class members are public (like C++):
    • member functions are virtual
    • member function is declared with explicity first argument representing the object (provided implicitly by the call).
  • built-in types can be used as base classes for extension (unlike C++).
  • most built-in oeprators can be redefined (like C++)

9.1 Names and Objects:

aliasing: multiple names (in multiple scopres) can be bound to the same object:

  • immuatable basic types (numbers, strings, tuples) saftely ignore affect
  • mutable objects (lists, dictionaries, etc): aliases behave like pointers
  • E.g. passing an object is cheap since only a pointer is passed by the implementation.

9.2 Python Scopes and Namespaces

9.2.1 namespace

  • mapping from names to objects, examples:
  • built-in names (inc functions e.g. ab())
  • built-in exception names;
  • global names in a module;
  • local names in a function invocation.

9.2.2 different namespaces

  • There is no relation between names in different namespaces; e.g. two different modules may both define a function maximize without confusion
  • users must prefix with the module name.

9.2.3 attribute

  • any name following a dot: e.g. z.real, real is an attribute of object z.

9.2.4 attribute references

  • modname.funcname: modname is a module object and funcname is an attribute of it.

9.2.5 attributes are read-only or writable

  • attributes assignment is possible: modname.the_answer = 42
  • attributes can be deleted: del modname.the_answer

9.3 Scopes and Namespaces Example

def scope_test():
    def do_local():
        spam = "local spam"

    def do_nonlocal():

9.5 Instance Objects

Instance objects only understand attribute references, there are two kinds:

  • data attributes: exist when first assigned x.counter = 1
  • methods: a function that ‚Äúbelongs to‚Äù an object

9.5.1 Method Objects

By definition, all attributes of a class that are function objects define corresponding methods of its instances, for example:

  • Since MyClass.f is a function (function object), then x.f is a valid method reference
  • But Myclass.i is not a function, therefore x.i is not.
    • Note 1: x.f is a method object (not function object)
    • Note 2: MyClass.f is a function object
class MyClass:
    """A simple example class"""
    i = 12345
    
    def __init__(self):
        self.data = []

    def f(self): # this method defined in a class is not associated with any object instance yet
        return 'hello world'
    
x = MyClass()

f (unbound method):

  • is just a function belonging to the class.

x.f (bound method object):

  • when method is accessed through object instance,
  • Python performs ‚Äúbinding‚Äù
    • binding process involves passing the instance as the first argument (‚Äúself‚Äù) to method
  • the method is tied to specific object instance
    • instance attributes and state become accessible to the method.
bound_method = x.f  # bind method to x object instance
bound_method        # bound_method
bound_method()      # call the bound_method, implicity self
unbound_method = MyClass.f
unbound_method      # uncommon way to use it
unbound_method(x) # method access direclty from class, requires instance explicitly pass

9.5 Class and Instance Variables

Instance Variables: data unique to each instance Class Variables: attributes and methods shared by all instances (of class)

class Dog:
    kind = 'canine'         # class variable shared by all instances
    def __init__(self, name):
        self.name = name    # instance variable unique to each instance

d = Dog('Fido')
e = Dog('Buddy')
print(d.kind)                  # shared by all dogs
print(e.kind)                  # shared by all dogs
print(d.name)                  # unique to d
print(e.name)                  # unique to e

shared data can have possibly surprising effects with involving mutable objects such as lists and dictionaries. For example, the tricks list in the following code should not be used as a class variable because just a single list would be shared by all Dog instances:

class Bag():
    def __init__(self) -> None:
        self._items = []

    def show_items(self,):
        print(self._items)
        
    def add_items(self,x):
        self._items.append(x)
    
    def add_items_twice(self,x):
        self.add_items(x)
        self.add_items(x)

    def add_items_n_times(self,x,n_times:int):
        for n in range(n_times): 
            self.add_items(x)

9.6 Inheritance:

isinstance() - Checking instance‚Äôs type:

  • isinstance(obj, int) will be True only if **obj.__class__ is int** (or some class derived from int)

issubclass() - Checking class inheritance:

  • issubclass(bool, int) is True since bool is a subclass of int
  • However, issubclass(float, int) is False since float is not a subclass of int.
