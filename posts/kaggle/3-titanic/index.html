2.1 Dealing with NaNs

To provide a prediction, each column is multiplied by a coefficient, NaNs can’t be multiplied by coefficients.

df = pd.read_csv(path/'train.csv')
df_og_keys = df.keys()
if PrintInfo: print(f"\n   BEFORE: \n{df.isna().sum()}") # True or False if NA
 # Each Nan's is equivalent to 1, sum all per col
top_modes = df.mode().iloc[0] # get first row of modes table per col (ties are random)
df.fillna(top_modes, inplace=True) # using the first row (a panda series with labels replace nas)

3.1 Numeric Values

  • Values too large (e.g. Fare $500) relatively to will others (e.g. )

Fare and Age have quite larger ranges compared to the other columns. Multiplying them by a coefficient will result is them dominating the function. Must scale or normalise these columns so their range is like the others.

4. Tensors

  • Create Independent variables: predictors (all continuous variables)
  • Create Dependent variables: target, i.e. Survived (variable to be predicted).

All model variables must be continuous variables.

They must be converted Pytorch Tensors (in order to use the special tensor methods):

1D Tensor: is like an Array
2D Tensor: is like a Table/Matrix
3D Tensor: is 3-Dimensional and so forth.

4.1 Dependent Variable

In this dataset, it is a single column/tensor, predicting whether a passenger is 1 (Survived) or 0 (Died).

from torch import tensor
dep_var_tensor_1D = tensor(df.Survived)
dep_var_tensor_1D[:3] # Like a numpy array, can get index to get items, first 3 here.

Stacking multiple tensors (columns) together creates our 2D-Tensor.
